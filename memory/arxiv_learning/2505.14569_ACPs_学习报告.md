## 摘要

AI代理已经越来越擅长执行复杂任务，例如编码、推理和多模态理解。然而，构建通才系统需要超越单一代理，转向集体推断——即通过结构化通信与协作互补的多样化、特定任务的专业多代理系统的范式。

目前，协调通常依赖于不精确且临时性的自然语言处理，这限制了复杂交互并阻碍了与特定领域专业代理之间的互操作性。

我们介绍了一种名为"代理上下文协议"（Agent Context Protocols, ACPs）的结构化通信方案：一种无域和无代理偏见的协议族，用于代理间的通信、协调及错误处理。

ACPs结合了：
1. **持久执行蓝图**——存储中间代理输出的显式依赖图
2. **标准化消息模式**

从而实现稳健且容错的多代理集体推理。

使用ACP增强的通才系统达到了行业领先的表现：
- 在AssistantBench上实现了28.3%的长时段网络协助准确率
- 生成了业界顶级水平的多模态技术报告
- 在人类评估中超越了商业AI系统

ACP高度模块化且可扩展，使从业者能够快速构建顶尖级通用代理。

## 核心概念

### 1. 集体推断（Collective Inference）

**定义**: 多智能体系统通过结构化通信与协作互补的范式

**与传统单智能体的区别**：
- 单智能体系统：一个代理处理所有任务
- 集体推断：多样化、特定任务的专业代理协作

### 2. Agent Context Protocols (ACPs)

**核心组成部分**：

#### 2.1 持久执行蓝图（Persistent Execution Blueprints）

- **显式依赖图**（Explicit Dependency Graphs）
- **存储中间代理输出**
- **容错机制**

#### 2.2 标准化消息模式（Standardized Message Schemas）

- **结构化通信**
- **领域无关**
- **代理无关**

### 3. 问题背景：自然语言协调的局限性

**当前问题**：
- 不精确（Imprecise）
- 临时性（Ad-hoc）
- 限制复杂交互
- 阻碍互操作性

**对比ACPs**：
| 维度 | 自然语言协调 | ACPs |
|-----|------------|------|
| 精确性 | 低 | 高（标准化模式） |
| 可扩展性 | 有限 | 高（模块化设计） |
| 容错性 | 差 | 好（依赖图+中间输出） |
| 互操作性 | 低 | 高（领域/代理无关） |

## 混沌理论解读

### 1. 从混沌系统到有序系统

**当前MAS状态（基于Interaction Theater论文）**：
- 个体层面：高熵（好）
- 系统层面：零耦合（坏）
- 结果：无序态（Disordered Phase）

**ACPs的作用**：
- 引入**显式依赖图** → 增加耦合
- **标准化消息模式** → 减少噪声
- **持久执行蓝图** → 建立反馈循环

**系统相变**：
```
无序态 → 边缘混沌 → 有序态
   ↓         ↓          ↓
零耦合    最优耦合    过度耦合
自然语言    ACPs      过度协议
```

### 2. 熵减机制

**个体熵**（保持高）：
- 每个代理保持任务专业化
- 高自我NCD（Normalized Compression Distance）

**系统熵**（通过ACPs降低）：
- 共享依赖图 → 减少系统不确定性
- 标准化消息 → 降低通信噪声
- 容错机制 → 提高系统稳定性

**互信息最大化**：
```
I(A, B) = H(A) + H(B) - H(A, B)
```
- H(A), H(B)（个体熵）: 保持高
- H(A, B)（联合熵）: 通过ACPs降低
- I(A, B)（互信息）: 最大化

### 3. 控制论应用

**开环系统（自然语言协调）**：
- 无反馈
- 无状态记忆
- 容错性差

**闭环系统（ACPs）**：
- 持久执行蓝图 = 状态记忆
- 依赖图 = 反馈机制
- 标准化消息 = 可预测的输入输出

## 关键洞察

1. **自发秩序需要结构化协议**
   - Interaction Theater证明了自发秩序不会自然涌现
   - ACPs提供了实现自发秩序的具体机制

2. **集体推断是MAS的未来**
   - 从单智能体 → 集体推断
   - 自然语言协调 → 结构化协议

3. **耦合混沌系统理论可以指导ACPs设计**
   - 边缘混沌作为设计目标
   - 耦合强度优化
   - 熵减机制设计

4. **ACPs实现了"被授权的自主性"**
   - 代理有自主性（专业化）
   - 但通过协议协调（共享状态）
   - 动态平衡（控制论）
